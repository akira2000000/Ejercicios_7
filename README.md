# Ejercicios_7
## Angélica Trejo Gálvez
## Licenciatura en Artes y Comunicación Digitales

El objetivo principal de este repositorio es reunir, organizar y documentar el proceso de trabajo realizado a lo largo de la UEA, ofreciendo un espacio centralizado donde se concentran todos los ejercicios, proyectos y materiales de referencia utilizados durante el curso. Este repositorio busca funcionar como un archivo digital estructurado que permita dar seguimiento al desarrollo de habilidades técnicas y conceptuales adquiridas, así como registrar las fases de producción de cada actividad. De esta forma, sirve tanto como evidencia académica como una herramienta de consulta futura, ya que facilita revisar versiones anteriores, comparar avances y comprender mejor la evolución de cada proyecto.

Además, el repositorio funciona como un archivo que registra el proceso de experimentación: desde entender la estructura básica de un sketch en p5.js, hasta construir funciones que interactúan con tablas, autenticación y APIs de Supabase. El propósito es que este repositorio sea útil no solo durante el curso, sino también en futuros proyectos de arte digital que necesiten recopilar información de usuarios, generar sistemas interactivos basados en datos o construir piezas conectadas a la nube.


La organización del repositorio está diseñada para que cualquier persona pueda encontrar rápidamente los ejemplos, ejercicios y proyectos relacionados con p5.js y Supabase. En la raíz se encuentra el archivo README, que explica el propósito general y guía al lector a través de la estructura interna. la primera carpeta es acerca de su uncionamiento y ejercicios basicos para introducir a p5, y las otras dos carpetas son sobre como çargar texto y en algunas imagenes en una pagina web utilizando Supabase. Se encuentran algunas anotaciones de clase dentro del código. Son ejercicios sencillos que si funcionan.


Crear y gestionar este repositorio ha sido una experiencia que me permitió entender mejor cómo se relacionan el arte digital y las herramientas de desarrollo web modernas. Al principio, integrar una base de datos a un sketch parecía algo complejo, pero conforme fui probando ejemplos y organizando el material dentro del repositorio, pude ver con claridad cómo estos sistemas se complementan. Documentar cada avance me ayudó a no perderme entre pruebas, errores y nuevas ideas, y a tener un registro claro de qué funcionaba y qué necesitaba ajustes.

Además, trabajar con GitHub me permitió adoptar prácticas más profesionales, como usar control de versiones, mantener carpetas bien organizadas y escribir documentación que haga sentido para otros. Aprendí que gestionar un repositorio no es solo guardar archivos, sino construir un espacio que refleje mi proceso de aprendizaje y mis decisiones técnicas.
